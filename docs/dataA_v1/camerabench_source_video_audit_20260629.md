### 对 Data A 的含义

- `1920×1080 → 1280×720` 是等比例缩放，覆盖 931 条主流 16:9 视频；加上原生 `1280×720` 后，至少 1,040 条（74.18%）可直接进入 landscape 16:9 profile。
- 宽画幅视频至少包括 `1920×960` 的 147 条与 `1920×818` 的 68 条。对它们直接 `scale=1280:720` 会改变几何比例，可能成为非编辑区域的额外分布偏移。
- portrait 视频只有 10 条；Data A v1 可先不纳入 VACE 主线，后续再考虑独立 portrait profile。
- 低分辨率与宽画幅视频不应优先用于小目标、文字、精细手部或人脸编辑。

## 5. 当前建议：生成规格与筛选顺序

| 决策项 | 建议 |
|---|---|
| 默认 VACE 帧数 | `49 frames` 优先 |
| `65 / 81 frames` | 只用于 SAM3 track 连续可见时间足够、且样本价值较高的 case |
| canonical fps | 维持 16 fps，同时保存 source timestamp 映射 |
| 默认 landscape profile | 16:9 等比例缩放到 1280×720 |
| 非 16:9 宽画幅 | 后续单独实现保持比例的 crop / padding policy；不要直接拉伸 |
| portrait | v1 主线暂缓 |
| 下一项审计 | SAM3 track 的连续可见时长、目标面积、遮挡与类别分布 |
